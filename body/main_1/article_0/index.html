What is j-make?

j-make is a lightweight client-side JavaScript library that reads a JSON array to build your page's DOM from <body> down, then fetches content for each element from a matching directory on your server. No build step. No back-end required. Works on GitHub Pages out of the box.

How it works

  1. j-make reads body.json on DOMContentLoaded.
  2. Each string in the array becomes an HTML element appended to <body> (or its parent for nested arrays).
  3. Every element gets a data-key (tagname_index) and a data-path derived from its position in the tree.
  4. j-make fetches index.html from the data-path directory and injects it into the element.
  5. Nested arrays produce nested elements. Content is prepended so it always appears before child elements.

Element content — this page as an example

Whatever is inside an element's index.html becomes that element's content. There are no special tags or templates — it is just plain HTML fetched and injected as-is.

The navigation bar you see at the top of this page is a direct example. The <header> element is created by j-make from body.json, and its content — the logo, version badge, and every nav link — comes entirely from body/header_0/index.html: 

<!-- body/header_0/index.html -->
<h1>j-make <span class="version">v1.0.0</span></h1>
<nav>
    <a href="#intro">About</a>
    <a href="#install">Install</a>
    <a href="#setup">Setup</a>
    <a href="#structure">Structure</a>
    <a href="https://github.com/richardkentgates/j-make/wiki">Wiki</a>
    <a href="https://github.com/richardkentgates/j-make">GitHub</a>
</nav>

Similarly, the sidebar cards come from body/main_1/aside_1/section_0/index.html, section_1/index.html, and section_2/index.html. This entire article is the content of body/main_1/article_0/index.html. Every piece of visible content on this page lives in its own directory — none of it is in index.html at the root.

Installation

Reference j-make from the jsDelivr CDN — no file to download or host:

<script src="https://cdn.jsdelivr.net/gh/richardkentgates/j-make@1.0.0/j-make.js"></script>

Pin to @master for the latest development version. jQuery 3.7.1+ must be loaded first.

index.html setup

Leave <body> completely empty — j-make populates it.

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My Site</title>
    <link rel="stylesheet" href="style.css">
    <script src="https://code.jquery.com/jquery-3.7.1.min.js"
            integrity="sha256-/JqT3SQfawRcv/BIHPThkBvs0OEvtFFmqPF/lYI/Cxo="
            crossorigin="anonymous"></script>
    <script src="https://cdn.jsdelivr.net/gh/richardkentgates/j-make@1.0.0/j-make.js"></script>
</head>
<body></body>
</html>

body.json

Place body.json at your site root. Strings become elements; nested arrays become children of the preceding element. Elements are indexed zero-based among their siblings. 

[
    "header",
    "main",
    [
        "article",
        "aside",
        [
            "section",
            "section"
        ]
    ],
    "footer"
]

Directory structure

Create a body/ folder at the root. For each element create a folder named {tagname}_{index} and place an index.html inside it. On GitHub Pages all index files must be .html. On a self-hosted server you can mix in .php or any other server-side language. 

index.html
body.json
body/
    header_0/
        index.html
    main_1/
        index.html
        article_0/
            index.html
        aside_1/
            index.html
            section_0/
                index.html
            section_1/
                index.html
    footer_2/
        index.html

Generated attributes

Every element j-make creates receives three attributes automatically:

AttributeValueExample
classAlways j-makeclass="j-make"
data-keytagname_indexdata-key="aside_1"
data-pathFull directory pathdata-path="body/main_1/aside_1"

Inspect these in DevTools to debug path or loading issues.
